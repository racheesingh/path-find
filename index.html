PathFind

Running traceroutes from arbitrary sources to arbitrary destinations is an expensive process. PathFind reuses existing measurements on RIPE Atlas to gather information about the Internet that users can subscribe to. The benefits are two-fold:

  • The users do not have to spend their own measurement budget to run traceroutes.
  • RIPE's nodes can be more efficiently utilised since redundant traceroutes will be avoided.

Interacting With PathFind

We have exposed a REST API for users to interact with PathFind. This is an easy to use (and work in progress) interface. As of now, our API:

  • Given an AS number (asn), returns the graph constituted by paths to to the asn based on recent data plane measurements.
  • Given two AS numbers (asn1, asn2), returns the path from asn1 to asn2 based on recent data plane measurements.
  • Provides an endpoint to get daily updates made to the complete AS graph based on the measurements done in the last 24 hours. ### Developement As you can see, PathFind is under active development. We hope to improve it incrementally based on how it gets used by us and the community.
